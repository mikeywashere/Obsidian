## Assignment

Create Obsidian.Api REST backend for Minecraft Bedrock server management. Implement ServerManager service and API controllers for server lifecycle and properties management.

## Deliverables

✅ **Obsidian.Api Project**
- New ASP.NET Core Web API project targeting .NET 9
- Added to Obsidian.sln

✅ **Infrastructure**
- Program.cs with CORS, SignalR, Swagger, DI configuration
- IServerManager interface for testability

✅ **ServerManager Service**
- Singleton ConcurrentDictionary<string, ManagedServer> for thread-safe state
- Cross-platform process launching (Windows: bedrock_server.exe, Linux/macOS: bedrock_server)
- Full stdout/stderr capture via event handlers
- Graceful shutdown with 10s timeout then force kill
- In-memory log buffer with configurable retention
- Demo server seeding on startup

✅ **API Controllers**
- ServersController: GET /api/servers, GET /api/servers/{id}, GET /api/servers/{id}/logs, POST /api/servers/{id}/start, POST /api/servers/{id}/stop
- PropertiesController: GET/PUT /api/servers/{id}/properties for server.properties editing

✅ **Technical Resolution**
- Fixed LogLevel namespace collision (Microsoft.Extensions.Logging vs Obsidian.Models)
- Made ServerProperties public in Obsidian project for API access
